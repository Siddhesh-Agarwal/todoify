# Todoify App Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a bootable TanStack Start app on Cloudflare Workers with D1 + Drizzle schema + better-auth (email/password) and protected routes — the foundation all task features build on.

**Architecture:** Hand-wire from the official TanStack Start "build-from-scratch" + Cloudflare "hosting" guides (exact configs verified against docs). Single Cloudflare Worker via `@cloudflare/vite-plugin`; data access through server functions only (no REST). Drizzle ORM over D1; better-auth instantiated per-request via `better-auth-cloudflare`'s `createAuth(env)` factory using the `cloudflare:workers` `env` binding.

**Tech Stack:** `@tanstack/react-start@1.168.27`, `@tanstack/react-router@1.170.17`, React 19, Vite, `@cloudflare/vite-plugin`, `drizzle-orm@0.45.2` + `drizzle-kit@0.31.10`, `better-auth@1.6.23` + `better-auth-cloudflare@0.3.0` + `@better-auth/drizzle-adapter`, shadcn/ui + Tailwind v4, react-hook-form + Zod, TanStack Query, vitest.

**Git note (per user):** `git init` + `.gitignore` happen in Task 2. **This plan contains NO commit steps** — the user chose "init git, don't commit." Commit only when the user explicitly asks. Each task ends with a Verify checkpoint instead.

**Source-dir deviation:** TanStack Start's scaffold, every official example, the build-from-scratch guide, and the hosting guide all use `src/` with `tanstackStart()` defaults. There is no documented way to point Start's router at `app/`. This plan uses `src/` and preserves AGENTS.md's sub-structure under it (`src/routes`, `src/components`, `src/lib/db`, `src/lib/auth`, `src/lib/schemas`, `src/server`). AGENTS.md's layout block reads `app/` — recommend updating it to `src/` for consistency (or relocating later). Easy to veto.

- [ ] **Step 1: Write `src/lib/auth/index.ts`** (per-request `createAuth(env)` factory — never called at module scope with bindings; dual-purpose: `auth` const for CLI schema generation uses a dummy adapter)

```ts
import { betterAuth } from 'better-auth'
import { withCloudflare } from 'better-auth-cloudflare'
import { drizzleAdapter } from '@better-auth/drizzle-adapter'
import { tanstackStartCookies } from 'better-auth/tanstack-start'
import { drizzle } from 'drizzle-orm/d1'
import type { IncomingRequestCfProperties } from '@cloudflare/workers-types'
import * as schema from '@/lib/db'

/**
 * Build the better-auth instance for the current request.
 * D1 binding only exists inside the request handler, so this MUST be called
 * per-request with the live env (never at module scope with bindings).
 */
export function createAuth(env?: CloudflareBindings, cf?: IncomingRequestCfProperties) {
  const db = env ? drizzle(env.DB, { schema }) : ({} as never)
  return betterAuth({
    baseURL: env?.BETTER_AUTH_URL ?? 'http://localhost:3000',
    ...withCloudflare(
      {
        autoDetectIpAddress: true,
        geolocationTracking: true,
        cf: cf ?? {},
        d1: env ? { db, options: { usePlural: true, debugLogs: false } } : undefined,
      },
      { emailAndPassword: { enabled: true } },
    ),
    ...(env
      ? {}
      : { database: drizzleAdapter({} as never, { provider: 'sqlite', usePlural: true }) }),
    plugins: [tanstackStartCookies()],
  })
}

// CLI-only export (no env): used by `@better-auth/cli generate` to emit auth.schema.ts.
// `usePlural: true` => tables are `users`, `sessions`, `accounts`, `verifications`.
export const auth = createAuth()

export type Session = Awaited<ReturnType<typeof auth.api.getSession>>
```

## Task 13: Protected home route + logout

**Files:**
- Modify: `src/routes/index.tsx`

- [ ] **Step 1: Replace `src/routes/index.tsx`** with a session-guarded home; `beforeLoad` runs server-side, builds `createAuth(env)`, redirects to `/login` when there's no session; logout uses better-auth's `signOut` (no custom server fn needed)

```tsx
import { createFileRoute, redirect, useNavigate } from '@tanstack/react-router'
import { createAuth } from '@/lib/auth'
import { authClient } from '@/lib/auth/client'
import { env } from 'cloudflare:workers'
import { Button } from '@/components/ui/button'

export const Route = createFileRoute('/')({
  beforeLoad: async ({ request }) => {
    const auth = createAuth(env)
    const session = await auth.api.getSession({ headers: request.headers })
    if (!session) throw redirect({ to: '/login' })
    return { session }
  },
  component: HomePage,
})

function HomePage() {
  const { session } = Route.useRouteContext()
  const navigate = useNavigate()

  async function logout() {
    await authClient.signOut()
    await navigate({ to: '/login' })
  }

  return (
    <div className="flex min-h-svh flex-col items-center justify-center gap-4 p-4">
      <h1 className="text-2xl font-semibold">Todoify</h1>
      <p className="text-muted-foreground">Signed in as {session?.user?.email}</p>
      <Button variant="outline" onClick={logout}>Log out</Button>
    </div>
  )
}
```
